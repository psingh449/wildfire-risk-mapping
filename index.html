Why you may see “ESTIMATED”
  • Some ACS tables come back as all-null at block group for a county.
  • When that happens, we use tract-level values and assign them to block groups in that tract.
  • The UI marks those as * and, in debug mode, tags like [est:tract].
Data source
  • [src:census]: value comes from decennial Census inputs (e.g., population / housing baselines).
  • [src:acs]: value comes from ACS tables (e.g., median home value, poverty, elderly, vehicle access).
  • [src:whp]: hazard signal sourced from wildfire probability / hazard products (USFS/related).
  • [src:hifld]: access-distance inputs sourced from HIFLD (e.g., hospitals, fire stations).
  • [src:osm]: access/connectivity proxy sourced from OpenStreetMap-derived features (roads / connectivity).
  • [px:…]: “proxy path” used when a primary dataset is missing at build time (still informative, but flagged).
  • [est:county-mean]: county-mean imputation used (e.g., missing block-group median).
  • [est:tract]: tract-level value assigned down to block groups when block-group ACS is missing.
  • [missing]: both primary and proxy paths are missing → shown as —.
Alpine County has a single 2020 Census block group at our chosen geography, so it appears as one polygon; within-county normalization uses a neutral value when there is no variation.
